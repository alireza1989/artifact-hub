# Artifact Hub — Demo Walkthrough

A ~10-minute script demonstrating the full loop: publish → browse → share → external review → AI synthesis → conversational MCP — plus the admin console. Each act lists the actions, what to point at, and the fallback if something is slow. Use it as a recording checklist or a live-demo script.

## Pre-demo checklist (10 minutes before)

- [ ] `pnpm db:seed --reset` against prod — clean demo catalog (8 artifacts across every kind, comment sets that trigger synthesis, one active share link), test noise swept.
- [ ] Production URL loads; gallery shows live previews (HTML cards render scaled pages, not icons).
- [ ] `pnpm smoke` green against production.
- [ ] Claude Desktop connected to `https://<host>/api/mcp` with the bearer token (Settings → Connectors — or copy it from the app's `/connect` page). Verify the 8 tools appear.
- [ ] Browser: logged **out** window (external-reviewer view) + logged **in** window (owner view, `/unlock` done). Remember the session cookie is per-hostname.
- [ ] Have a small HTML file (or snippet to paste) ready to publish — something visual, e.g. a pricing-page mockup.
- [ ] `/admin/ai` open in a tab — you'll return to it after the AI moments.

## Act 1 — First impression (1 min)

1. Open the gallery, logged out.
2. Point at: live per-kind previews (HTML pages actually rendering in cards, PDF first pages, text snippets), tag chips, kind filter, the clean visual identity.
3. Say: *"Everything here was generated by AI tools somewhere — this is where it stops being files in a bucket and becomes a catalog."*

## Act 2 — Publish with invisible AI (2 min)

1. Logged-in window → **Publish**. Drag the HTML file in. **Leave title, description, and tags blank.**
2. Publish → land on the artifact page.
3. Point at: title/description/tags filled, each with a subtle **"suggested"** badge. Edit the title → badge disappears (the stored value no longer matches the recorded suggestion).
4. Point at: the sandboxed live preview, the **Source** tab, the metadata sidebar.
5. Say: *"The AI filled what I skipped — but publish never waits on it, and if the model is down the filename becomes the title. AI as infrastructure, not as a feature."*
6. Optional beat: click **Re-run AI suggestions** to show regeneration under the same badge mechanism.

## Act 3 — Share + external anchored review (2.5 min)

1. On the artifact page: **Share links** → create a 72h link. Point at the one-time reveal copy ("won't be shown again — only a hash is stored").
2. Optional: paste the link into Slack — it unfurls with a branded card. *"Crawler fetches don't count as views."*
3. Open the link in the logged-out window. Point at: read-only view, visible expiry, no nav into the app.
4. **Select a sentence** in the preview → "Comment on this" → post a comment as an external reviewer. Point at the quote chip; click it → the passage scrolls into view and flashes.
5. (If demoing on an image artifact: click a spot → numbered pin.)
6. Add a second comment (different name) so the artifact reaches the synthesis threshold.
7. Back in the owner window: revoke the link with one click → reload the share tab → friendly "turned off" page, not a 404.

## Act 4 — AI feedback synthesis (1.5 min)

1. Owner window → the artifact page. The **Feedback summary** card is now there: consensus, disagreements, action items, sentiment.
2. Point at: every bullet cites the comments it came from — click a citation, it jumps to the comment. *"Traceable, not vibes."*
3. Point at the anchored comment's quote flowing into the summary context.
4. Say: *"It regenerates lazily when comments change, single-flighted so concurrent readers don't double-spend."*

## Act 5 — The MCP loop (2.5 min) ⭐

The centerpiece: the same platform, conversationally, from Claude Desktop.

1. Ask Claude: *"Publish this HTML to the artifact hub"* (paste a small page). Watch `publish_artifact` run → Claude reports the id and which fields the AI filled.
2. *"Now share it for 72 hours."* → `create_share_link` → a working URL. Open it in the browser — same share view as Act 3.
3. *"What's the feedback on the pricing mockup?"* → `search_artifacts` resolves the reference → `get_feedback` returns comments + the synthesis.
4. *"Any new artifacts this week?"* → `hub_stats` or `search_artifacts` with `since`.
5. Say: *"Web, REST, and MCP are three thin adapters over one core — this isn't a bot bolted on, it's the same code path."*
6. Refresh the gallery: the conversationally-published artifact is there, source-tagged `mcp`.

## Act 6 — NL search + admin (1.5 min)

1. Gallery search box: type **"html mockups with feedback from last week"** → results. Say: *"A model translates that into the filters the search index already supports — keyword queries never touch the model, and a bad translation falls back to plain search. Same box, no AI button."*
2. **Admin → AI**: the telemetry from everything just demoed — calls, cost, latency percentiles, outcomes. *"Every model call this demo made is a row here."*
3. **Admin → Share links**: the platform-wide inventory, including the revoked one from Act 3.
4. **Admin → Tags**: run **Suggest cleanup** → review the proposed merges → apply one. *"The model proposes, the owner disposes — it can only regroup tags it was shown."*

## Close (30 s)

- Gates: 252 automated tests, LLM eval harness at 100% schema validity and 100% injection resistance across 28 live-model fixtures, post-deploy smoke on every production deployment.
- One deployable, one core, three surfaces. Cuts documented with rationale in [WRITEUP.md](WRITEUP.md).

## Fallbacks

| If | Then |
|---|---|
| Haiku is slow/down during publish | The flow still completes (filename title) — narrate that this *is* the design |
| Claude Desktop won't connect | `/connect` page on screen + `pnpm smoke` output shows the MCP round-trip working |
| Synthesis card doesn't appear | It needs ≥2 comments; add one more, reload (regeneration is on-read) |
| NL search returns nothing | Say so honestly — it fell back to raw FTS, which is the invariant being demonstrated |
